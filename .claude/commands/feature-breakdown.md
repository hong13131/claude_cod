● # Task Decomposition Command

  ## Purpose
  큰 문제를 관리 가능한 작은 작업으로 세분화하는 체계적 분석 도구

  ## Usage
  ```bash
  /decompose "$ARGUMENT"

  Command Template

  1. 문제 분석 (Problem Analysis)

  핵심 문제: $ARGUMENT

  문제 분석:
  - 본질: [문제의 핵심 목표는 무엇인가?]
  - 범위: [포함/제외될 요소들은?]
  - 제약사항: [시간, 리소스, 기술적 제약은?]
  - 성공 기준: [완료를 어떻게 정의할 것인가?]

  2. 의존성 매핑 (Dependency Mapping)

  graph TD
      A[메인 태스크] --> B[하위 태스크 1]
      A --> C[하위 태스크 2]
      B --> D[세부 작업 1.1]
      B --> E[세부 작업 1.2]

  의존성 분석:
  - 선행 작업: [먼저 완료되어야 할 작업들]
  - 병렬 처리 가능: [동시에 진행할 수 있는 작업들]
  - 블로킹 요소: [다른 작업을 막을 수 있는 요소들]

  3. 작업 분해 (Task Breakdown)

  Level 1: 주요 기능/영역별 분해

  - [영역 1]: [설명]
  - [영역 2]: [설명]
  - [영역 3]: [설명]

  Level 2: 세부 작업 분해

  [영역 1]
  - [구체적 작업 1.1] - 예상 시간: [X시간]
  - [구체적 작업 1.2] - 예상 시간: [X시간]

  [영역 2]
  - [구체적 작업 2.1] - 예상 시간: [X시간]
  - [구체적 작업 2.2] - 예상 시간: [X시간]

  4. 우선순위 매트릭스 (Priority Matrix)

  | 작업    | 중요도             | 긴급도             | 리스크             | 우선순위     |
  |-------|-----------------|-----------------|-----------------|----------|
  | [작업명] | High/Medium/Low | High/Medium/Low | High/Medium/Low | P1/P2/P3 |

  5. 실행 계획 (Execution Plan)

  Phase 1: 기반 작업 (Foundation)

  기간: [X일]
  - [P1 작업들]
  - 완료 기준: [명확한 완료 조건]

  Phase 2: 핵심 개발 (Core Development)

  기간: [X일]
  - [P2 작업들]
  - 완료 기준: [명확한 완료 조건]

  Phase 3: 마무리 및 검증 (Finalization)

  기간: [X일]
  - [P3 작업들]
  - 완료 기준: [명확한 완료 조건]

  6. 리스크 관리 (Risk Management)

  | 리스크     | 확률              | 영향도             | 대응 방안     |
  |---------|-----------------|-----------------|-----------|
  | [리스크 1] | High/Medium/Low | High/Medium/Low | [구체적 대응책] |

  7. 검증 체크리스트 (Validation Checklist)

  각 작업 완료 시 확인사항:
  - 요구사항 충족 여부
  - 테스트 통과 여부
  - 문서화 완료 여부
  - 코드 리뷰 완료 여부

  최종 완료 시 확인사항:
  - 전체 목표 달성 여부
  - 성능 기준 충족 여부
  - 사용자 수용 테스트 통과
  - 배포 준비 완료

  8. 모니터링 지표 (Monitoring Metrics)

  - 진행률: [완료된 작업 수 / 전체 작업 수]
  - 일정 준수율: [예정 시간 대비 실제 소요 시간]
  - 품질 지표: [테스트 통과율, 버그 발생률]
